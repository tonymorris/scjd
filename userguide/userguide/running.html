Running the Client Application

While the client application is running, certain actions may be performed using the client interface. The interface contains a tool bar, a set of menu options and a pop-up menu (on a data record only) from which these actions can be invoked. Each action has a short cut "accelerator" key(s) associated with it to alllow operation of the application with the keyboard. Each action may or may not be enabled depending upon the state of the client application. Action icons are usually "grayed-out" when the action is disabled - this is Look and Feel dependent.

About Dialog (F1)

This action brings up a dialog that displays a brief description about the application, click anywhere on the dialog, the main application window or the 'X' close button to close the About dialog. This action is always enabled.

Configuration Dialog (F9)

This action brings up a dialog that allows the user to configure the application. Although this action is always enabled, certain configuration attributes on the dialog will be "read-only" when the client application is connected to a data source. The user will not be able to use the main application window while this dialog is being displayed. Click 'OK' to apply the configuration changes or 'Cancel' to cancel any changes - this will close the Configuration Dialog. More information on configuration can be found in the Configuration section.

Connect/Disconnect / (Control-ENTER)

The connect/disconnect action will connect to the data source if the application is currently disconnected, or disconnect from the data source if the application is currently connected. The data source may be the local data file if the application is in local mode or it will be the remote Data object if the application is in networked mode. This action will always be enabled. See the Starting the Application section for more information on the client application modes.

Copy Data Record (Shift-C)

This action will copy the currently selected data record to the system clipboard. This action is only enabled when there is a data record selected and the application is connected to a data source, otherwise, it is disabled. The system clipboard contents can be used outside of the client application.

Create Data Record (Control-R)

This action will bring up a dialog to allow the entry of a new data record. This action is only enabled when the application is connected to a data source, otherwise, it is disabled. Press 'OK' to commit the new data record to the data source, press the 'Cancel' button to cancel the creation of a new data record.

Cut Data Record (Shift-X)

This action will remove the currently selected data record and copy its contents to the system clipboard. If the application is configured to confirm 'Cut data record' operations, a confirmation dialog will appear which will allow the user to cancel the operation if desired. This action is only enabled when there is a data record selected and the application is connected to a data source, otherwise, it is disabled. The system clipboard contents can be used outside of the client application.

Delete Data Record (Delete)

This action will deleted the currently selected data record from the data source. If the application is configured to confirm 'Delete data record' operations, a confirmation dialog will appear which will allow the user to cancel the operation if desired. This action is only enabled when there is a data record selected and the application is connected to a data source, otherwise, it is disabled.

Edit Data Record (Space)

This action will bring up a dialog to allow the editing of the currently selected data record. This action is only enabled when there is a data record selected and the application is connected to a data source, otherwise, it is disabled. Press 'OK' to commit the edited data record to the data source, press the 'Cancel' button to cancel the editing of the selected data record. This will close the dialog.

Exit the Application (Control-Q)

This action will disconnect from the data source (if it is connected) before exiting the application. If the application is configured to confirm 'Exit application' operations, a confirmation dialog will appear which will allow the user to cancel the operation if desired. This action is always enabled.

Paste Data Record (Shift-V)

This action will bring up a dialog that contains data from the system clipboard to allow the user to create a new data record from the clipboard contents. If the system clipboard does not contain valid record data, an error will be reported by the application. This action is only enabled when the application is connected to a data source, otherwise, it is disabled. Press 'OK' to commit the pasted data record to the data source, press the 'Cancel' button to cancel the creation of the pasted data record.

Refresh All Data Records Display (F5)

This action will refresh the display of data records to show all data records from the data source. This action is only enabled when the application is connected to a data source, otherwise, it is disabled.

Refresh Current Display (F7)

This action will refresh with an updated display of data records. If the data record display contains a subset of the total data records in the data source (from a Search), then only the subset will be updated in the display, otherwise, all records will be updated. This action is only enabled when the application is connected to a data source, otherwise, it is disabled.

Search Data Records (Control-F)

This action will bring up a dialog to allow the user to enter search criteria for data records. Data records are matched if they start with the given String criteria unless the user specifies to match fields exactly with the check box. The search criteria is case-sensitive. Once the search has been performed, only the data records from the data source that match the given criteria will be displayed. This action is only enabled when the application is connected to a data source, otherwise, it is disabled.
