Troubleshooting

Problem:
The client application is throwing a java.lang.IllegalStateException.

Solution:
This problem is the result of the data server being in an invalid state. Ensure that the data server is using a valid data file that contains valid header data.

Problem:
The server application is reporting the following exception: "java.io.FileNotFoundException: Data source file name not specified"

Solution:
The server application is not configured with a data file. Use the server dialog application to write a configuration file to configure the location of the server data file.

Problem:
The server application is reporting the following exception: "java.io.FileNotFoundException: Specified data file name does not exist: <filename>"

Solution:
The server application cannot find the data file (which is given as the <filename> in the error message). Use the server dialog application to configure the correct location of the server data file.

Problem:
Once the client is connected to the data source, no data records are displayed even though there are several data records in the server data file.

Solution:
The data schema is not configured correctly in the client application. Make sure that the configured client schema and the schema of the data file are consistent.

Problem:
After data records are added with the client application, a refresh causes them to disappear (or some other similar odd behaviour).

Solution:
The data schema is not configured correctly in the client application. Make sure that the configured client schema and the schema of the data file are consistent. An example of a correctly configured data schema for the data server is provided here. The default configuration settings provide this same setting.

Problem:
The client application is reporting that the currently selected data record has been deleted. Why are deleted records being displayed ?

Solution:
Another user who is also connected to the same server has deleted the data record and the client application needs to be refreshed to reflect the changes.

Problem:
The client application cannot connect successfully to the remote data server.

Solution:
Make sure that both the client and server able to "see" each other over the network (no Network Address Translation) and that there is no firewall blocking port 1099 or any other ports used by Remote Method Invocation (RMI).
