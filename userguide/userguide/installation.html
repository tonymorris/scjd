Installation

The installation begins with obtaining the single Java Archive (JAR) file which makes up the entire assignment submission. This file is named scjd-bxxxx.jar where xxxx is the build release number of the submission file. This file contains all files required to perform the installation.

Package Contents

scjd-bxxxx.jar
---- code
---- docs
-------- javadoc
-------- choices.txt
-------- instructions.html
-------- userguide.html
---- runme.jar
---- db-2x2.db
---- version.txt

"scjd-bxxxx.jar" is the top level submission file where xxxx is the build release number of the assignment submission file.
"code" is a directory containing all the source code that makes up the assignment submission.
"docs" is a directory containing documentation that is relevant to the assignment submission.
"javadoc" is a directory containing JavaDoc documentation for all the source code that makes up this assignment submission.
"choices.txt" is an ASCII text file that describes design decisions made during development of the application.
"instructions.html" is an HTML file that was part of the assignment download - this file remains unchanged.
"userguide.html" is one file of many HTML files that make up this user guide.
"runme.jar" is the single executable Java Archive (JAR) file that is required to execute the application.
"db-2x2.db" is a database file that was part of the assignment download - this file remains unchanged.
"version.txt" is an ASCII text file that specifies the development and test platform of the application.

System Requirements

The application requires a Java 2 Software Development Kit version 1.4 or higher in order to run. Memory requirements are that of the installed Java 2 SDK. The client machine must have a direct connection to the server machine (no Network Address Translation) and no firewall to prevent access over Remote Method Invocation (RMI).

Installing

The installation requires extraction of the single executable file and related documentation files from the assignment submission file. This can be achieved using the jar utility which comes with the Java 2 SDK 1.4. Once the executable file has been extracted, it can be executed from a command line session using the jar switch.

If the application is to be started with the server, the configured database file must contain valid header data (optionally, it may also contain data records). If this file does not exist, the server will report a java.io.FileNotFoundException at start up time.

For both the server and client part of the application, a file called "suncertify.properties" may optionally exist in the JVM working directory that specifies configuration parameters to the application. If this configuration file does exist, a set of default configuration properties is used. The default values have a data schema defined that is consistent with version 1.0 of the data server. More information on how to configure the application can be found in the Configuration section.
